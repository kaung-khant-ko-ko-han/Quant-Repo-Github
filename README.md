## နိဒါန်း

Quant trading (အရေအတွက်ဆိုင်ရာ ကုန်သွယ်မှု) လုပ်တယ်လို့ တစ်စုံတစ်ယောက်ပြောတာကြားဖူးရင်၊ spreadsheet (စာရင်းဇယား) နဲ့ ခန့်မှန်းမှုလို့ တွေးမိတာမျိုး ဖြစ်နိုင်ပေမယ့်၊ တကယ်တမ်းကျတော့ ဒါဟာ အဲဒီထက် အများကြီး စနစ်ကျတဲ့ သဘောတရားတစ်ခုပါ။ Quant trading ဆိုတာ စည်းမျဉ်းအခြေခံတဲ့ ကုန်သွယ်မှုဆိုင်ရာ ဆုံးဖြတ်ချက်တွေချဖို့ data (ဒေတာ)၊ statistics (စာရင်းအင်းပညာ) နဲ့ code (ကုဒ်) တွေကို အသုံးပြုတာဖြစ်ပါတယ်။

သင်ဟာ momentum (အရှိန်အဟုန်)၊ mean reversion (ပျမ်းမျှပြန်လည်ရောက်ရှိခြင်း) ဒါမှမဟုတ် pairs trading (အတွဲလိုက်ကုန်သွယ်မှု) လိုအိုင်ဒီယာတွေကိုယူပြီး၊ သူတို့ကို ရှင်းရှင်းလင်းလင်း သတ်မှတ်ထားတဲ့ မဟာဗျူဟာတွေအဖြစ် ပြောင်းလဲကာ၊ သမိုင်းဝင်ဒေတာတွေနဲ့ backtest (နောက်ပြန်စမ်းသပ်) ပြီးရင်၊ risk management (စွန့်စားမှုစီမံခန့်ခွဲမှု)၊ position sizing (ရာထူးအရွယ်အစားသတ်မှတ်ခြင်း) နဲ့ execution logic (လုပ်ဆောင်မှုယုတ္တိဗေဒ) တွေကို အလွှာလိုက်ထည့်သွင်းပါတယ်။ ရည်ရွယ်ချက်ကတော့ စိတ်ခံစားမှုနဲ့ ဓာတ်ပြုမှုမျိုးမဟုတ်ဘဲ စနစ်ကျပြီး တစ်သမတ်တည်းဖြစ်ဖို့ပါပဲ။

ဒီဆောင်းပါးမှာ၊ ကျွန်တော်တို့ဟာ strategies (မဟာဗျူဟာများ)၊ frameworks (မူဘောင်များ)၊ coding examples (ကုဒ်ရေးသားခြင်းနမူနာများ)၊ research tools (သုတေသနကိရိယာများ)၊ interview questions (အင်တာဗျူးမေးခွန်းများ)၊ curated resources (စီစဉ်ပေးထားသော အရင်းအမြစ်များ) နဲ့ practical guides (လက်တွေ့လမ်းညွှန်ချက်များ) တို့ကို လွှမ်းခြုံထားတဲ့ GitHub repository ၁၀ ခုကို ပြန်လည်သုံးသပ်သွားမှာပါ။ ဒါတွေအားလုံးပေါင်းစပ်ခြင်းအားဖြင့်၊ အစပြုသူရဲ့ စမ်းသပ်မှုတွေကနေ ပိုမိုလေးနက်တဲ့ quantitative trading systems (အရေအတွက်ဆိုင်ရာ ကုန်သွယ်မှုစနစ်များ) ဆီကို ကြီးထွားလာဖို့ လိုအပ်တဲ့ domains (ဒိုမိန်းများ)၊ workflows (အလုပ်အသွားအလာများ) နဲ့ technical stack (နည်းပညာအစုအစည်း) တို့ကို သင်သိရှိစေနိုင်ပါတယ်။

**ငြင်းဆိုချက်-** ဤအကြောင်းအရာသည် ပညာရေးဆိုင်ရာရည်ရွယ်ချက်အတွက်သာဖြစ်ပြီး ဘဏ္ဍာရေးဆိုင်ရာ အကြံပေးချက်မဟုတ်ပါ။

## Quantitative Trading ကိုကျွမ်းကျင်ရန် GitHub Repositories များ

#### // 1. Python Quant Trading Strategies (Python အရေအတွက်ဆိုင်ရာ ကုန်သွယ်မှု မဟာဗျူဟာများ)

[Python Quant Trading Strategies](https://github.com/je-suis-tm/quant-trading) repository မှာ RSI, Bollinger Bands, MACD, pairs trading, options straddles နဲ့ Monte Carlo simulations အပါအဝင် Python strategy နမူနာများစွာ ပါဝင်ပါတယ်။ ကုန်သွယ်မှုဆိုင်ရာ အိုင်ဒီယာတွေကို လုပ်ဆောင်နိုင်တဲ့ code အဖြစ် ဘယ်လိုပြောင်းလဲသနည်းဆိုတာကို နားလည်ဖို့ အထူးသဖြင့် အသုံးဝင်ပါတယ်။

သင်ဟာ quant trading အတွက် အသစ်ဖြစ်ပါက၊ မဟာဗျူဟာတွေကို ဘယ်လိုတည်ဆောက်ပြီး အကဲဖြတ်ရမလဲဆိုတာ လေ့လာဖို့ ဒါဟာ လက်တွေ့ကျတဲ့ အစပျိုးအချက်တစ်ခုဖြစ်ပါတယ်။

#### // 2. StockSharp

[StockSharp](https://github.com/StockSharp/StockSharp) ဟာ equity (ရှယ်ယာ)၊ futures (အနာဂတ်ကုန်စည်)၊ options (ရွေးချယ်မှု) နဲ့ crypto (ခရစ်တိုငွေကြေး) အပါအဝင် ပိုင်ဆိုင်မှုအမျိုးအစားများစွာတွင် live markets (တိုက်ရိုက်စျေးကွက်များ) နဲ့ ချိတ်ဆက်ပြီး ကုန်သွယ်ရေး robots တွေကို တည်ဆောက်ဖို့ ရင့်ကျက်တဲ့ platform (ပလက်ဖောင်း) တစ်ခုဖြစ်ပါတယ်။ ရိုးရိုး notebooks တွေနဲ့မတူဘဲ၊ ဒီ platform က production-level architecture (ထုတ်လုပ်မှုအဆင့်ဗိသုကာ)၊ connectors (ချိတ်ဆက်ကိရိယာများ)၊ order management (အမိန့်စီမံခန့်ခွဲမှု) နဲ့ live execution concepts (တိုက်ရိုက်လုပ်ဆောင်မှုသဘောတရားများ) တို့ကို သင်နဲ့ မိတ်ဆက်ပေးပါတယ်။

#### // 3. Riskfolio-Lib

[Riskfolio-Lib](https://github.com/dcajasn/Riskfolio-Lib) ဟာ portfolio optimization (အစုစုပိုမိုကောင်းမွန်အောင်ပြုလုပ်ခြင်း) နဲ့ risk modeling (စွန့်စားမှုပုံစံဖော်ခြင်း) တွေကို အဓိကထားပါတယ်။ ဒါတွေဟာ ကုန်သွယ်ရေးအချက်ပြမှုတွေကို ဖွဲ့စည်းတည်ဆောက်ထားတဲ့ ရင်းနှီးမြှုပ်နှံမှုဆိုင်ရာ ဆုံးဖြတ်ချက်တွေအဖြစ် ပြောင်းလဲဖို့အတွက် အလွန်အရေးကြီးပါတယ်။ Optimization frameworks (ပိုမိုကောင်းမွန်အောင်ပြုလုပ်ရေးမူဘောင်များ) သုံးပြီး strategic asset allocation (မဟာဗျူဟာကျသော ပိုင်ဆိုင်မှုခွဲဝေမှု) နဲ့ quantitative portfolio design (အရေအတွက်ဆိုင်ရာ အစုစုဒီဇိုင်း) အတွက် လက်တွေ့အကျဆုံး Python library တွေထဲက တစ်ခုဖြစ်ပါတယ်။

#### // 4. EliteQuant

[EliteQuant](https://github.com/EliteQuant/EliteQuant) ဟာ quantitative trading (အရေအတွက်ဆိုင်ရာ ကုန်သွယ်မှု) နဲ့ modeling (ပုံစံဖော်ခြင်း) အရင်းအမြစ်များကို စီစဉ်ပေးထားတဲ့ စုစည်းမှုတစ်ခုဖြစ်ပါတယ်။ ဒါဟာ ကုန်သွယ်မှုသဘောတရားများ၊ မော်ဒယ်လုပ်နည်းစနစ်များနဲ့ portfolio management (အစုစုစီမံခန့်ခွဲမှု) ခေါင်းစဉ်များအပါအဝင် ဖွဲ့စည်းတည်ဆောက်ထားတဲ့ သင်ယူမှုဆိုင်ရာ အကြောင်းအရာတွေကို ပံ့ပိုးပေးပါတယ်။ အရင်းအမြစ်များစွာတွင် ရှာဖွေနေချိန်မဖြုန်းတီးဘဲ ဘာတွေလေ့လာရမယ်ဆိုတဲ့ လမ်းပြမြေပုံတစ်ခု လိုချင်တဲ့အခါမျိုးမှာ ဒါဟာ အသုံးဝင်ပါတယ်။

#### // 5. Quant Developers Resources (Quant Developer အရင်းအမြစ်များ)

[Quant Developers Resources](https://github.com/cybergeekgyan/Quant-Developers-Resources) repository ဟာ quant developer (အရေအတွက်ဆိုင်ရာ ဆော့ဖ်ဝဲလ်အင်ဂျင်နီယာ)၊ quant researcher (အရေအတွက်ဆိုင်ရာ သုတေသီ) နဲ့ quant trader (အရေအတွက်ဆိုင်ရာ ကုန်သည်များ) အသက်မွေးဝမ်းကျောင်းလမ်းကြောင်းများကို အဓိကထားပါတယ်။ Interview preparation (အင်တာဗျူးကြိုတင်ပြင်ဆင်မှု) ခေါင်းစဉ်များ၊ အကြံပြုထားသော စာအုပ်များ၊ probability (ဖြစ်နိုင်ခြေသီအိုရီ) နဲ့ statistics (စာရင်းအင်းပညာ) ကိုးကားချက်များ၊ နဲ့ quant အခန်းကဏ္ဍများတွင် မျှော်လင့်ရတဲ့ programming skills (ပရိုဂရမ်းမင်းစွမ်းရည်များ) တို့ ပါဝင်ပါတယ်။ Quant အင်တာဗျူးတွေအတွက် ပြင်ဆင်နေတယ်ဆိုရင်၊ ဒီ repository က မင်းရဲ့ပြင်ဆင်မှုကို လုပ်ငန်းနယ်ပယ်ရဲ့ မျှော်လင့်ချက်များနဲ့ ကိုက်ညီအောင် ကူညီပေးပါတယ်。

#### // 6. TradeMaster

[TradeMaster](https://github.com/TradeMaster-NTU/TradeMaster) ဟာ reinforcement learning (အားဖြည့်သင်ကြားမှု) အခြေခံတဲ့ ကုန်သွယ်မှုဆိုင်ရာ အလုပ်အသွားအလာများအတွက် ဒီဇိုင်းထုတ်ထားတဲ့ open-source research platform (ပွင့်လင်းရင်းမြစ်သုတေသနပလက်ဖောင်း) တစ်ခုဖြစ်ပါတယ်။ Environment design (ပတ်ဝန်းကျင်ဒီဇိုင်း)၊ model training (မော်ဒယ်လေ့ကျင့်ခြင်း)၊ evaluation (အကဲဖြတ်ခြင်း) နဲ့ backtesting (နောက်ပြန်စမ်းသပ်ခြင်း) အပါအဝင် သုတေသနဘဝစက်ဝန်းတစ်ခုလုံးကို လွှမ်းခြုံထားပြီး၊ ကုန်သွယ်မှုအတွက် modern machine learning approaches (ခေတ်သစ်စက်သင်ယူမှုနည်းလမ်းများ) တွေကို စူးစမ်းရှာဖွေနေတယ်ဆိုရင် အဲဒါက အဖိုးတန်ပါတယ်။

#### // 7. Sunday Quant Scientist

[Sunday Quant Scientist](https://github.com/quant-science/sunday-quant-scientist) ဟာ quantitative analysis (အရေအတွက်ဆိုင်ရာ ခွဲခြမ်းစိတ်ဖြာမှု)၊ portfolio management (အစုစုစီမံခန့်ခွဲမှု) နဲ့ လက်တွေ့ကျတဲ့ ရင်းနှီးမြှုပ်နှံမှုသုတေသနတွေကို အဓိကထားတဲ့ newsletter-backed repository (သတင်းလွှာအခြေခံရီပိုစီတရီ) တစ်ခုဖြစ်ပါတယ်။ code ရေးတာထက် ကျော်လွန်တဲ့ ထိုးထွင်းသိမြင်မှုတွေနဲ့ context တွေကို လိုချင်တဲ့အခါ၊ အထူးသဖြင့် တစ်သမတ်တည်း သင်ယူမှုနဲ့ idea ထုတ်လုပ်မှုအတွက် အလွန်ကောင်းမွန်ပါတယ်။

#### // 8. QuantMuse

[QuantMuse](https://github.com/0xemmkty/QuantMuse) ဟာ real-time data processing (အချိန်နှင့်တစ်ပြေးညီ ဒေတာလုပ်ဆောင်ခြင်း)၊ analytics (ပိုင်းခြားစိတ်ဖြာခြင်း) နဲ့ risk management (စွန့်စားမှုစီမံခန့်ခွဲမှု) အစိတ်အပိုင်းတွေ အပါအဝင် ပိုမိုပြည့်စုံတဲ့ quantitative trading system (အရေအတွက်ဆိုင်ရာ ကုန်သွယ်မှုစနစ်) တစ်ခုကို တည်ဆောက်ခြင်းကို အဓိကထားပါတယ်။ သီးခြား scripts တွေအစား မတူညီတဲ့ modules တွေ ဘယ်လို ဖွဲ့စည်းတည်ဆောက်ထားတဲ့ ကုန်သွယ်မှုစနစ်တစ်ခုအဖြစ် အံဝင်ခွင်ကျဖြစ်သွားသလဲဆိုတာကို နားလည်ဖို့ ဒါက ကူညီပေးပါတယ်။

#### // 9. Options Trading Strategies in Python (Python ရှိ Options ကုန်သွယ်မှုမဟာဗျူဟာများ)

[Options Trading Strategies in Python](https://github.com/PyPatel/Options-Trading-Strategies-in-Python) repository ဟာ Python ထဲမှာ options strategy (ရွေးချယ်မှုမဟာဗျူဟာ) ဖွံ့ဖြိုးတိုးတက်မှုကို အထူးအလေးပေးပါတယ်။ Options payoff structures (ရွေးချယ်မှုအချောသတ်ဖွဲ့စည်းပုံ) တွေကို နားလည်ပြီး spreads (အသားတင်အမြတ်) နဲ့ straddles (စျေးနှုန်းနှစ်မျိုးလုံးကိုအခြေခံသည့် မဟာဗျူဟာ) လိုမျိုး မဟာဗျူဟာတွေကို code ထဲမှာ အကောင်အထည်ဖော်ချင်တယ်ဆိုရင် ဒါက အသုံးဝင်ပါတယ်။

#### // 10. Howtrader

[Howtrader](https://github.com/51bitquant/howtrader) ဟာ crypto-focused trading framework (ခရစ်တိုငွေကြေးကို အဓိကထားတဲ့ ကုန်သွယ်မှုမူဘောင်) တစ်ခုဖြစ်ပြီး strategy development (မဟာဗျူဟာဖွံ့ဖြိုးရေး)၊ backtesting (နောက်ပြန်စမ်းသပ်ခြင်း) နဲ့ live execution (တိုက်ရိုက်လုပ်ဆောင်ခြင်း) တွေကို ပံ့ပိုးပေးပါတယ်။ External signals (ပြင်ပအချက်ပြမှုများ) တွေကို ဘယ်လိုပေါင်းစည်းရမလဲ၊ ကုန်သွယ်မှု workflows တွေကို အလိုအလျောက်လုပ်ဆောင်ဖို့၊ နဲ့ crypto ecosystem (ခရစ်တိုငွေကြေးဂေဟစနစ်) အတွင်း exchange connectivity (ဖလှယ်မှုချိတ်ဆက်မှု) ကို ဘယ်လိုကိုင်တွယ်ရမလဲဆိုတာကို နားလည်ဖို့ အသုံးဝင်ပါတယ်။

## နိဂုံးချုပ် အတွေးအမြင်များ

ရိုးရိုးသားသားပြောရရင်၊ လူအများစုဟာ quant trading ကို နောက်ပြန်လှန် ချဉ်းကပ်ကြပါတယ်။ သူတို့ဟာ ပထမဆုံး မဟာဗျူဟာတစ်ခုကို ရှာကြပြီး နောက်မှသာ သူတို့အတွက် risk models (စွန့်စားမှုပုံစံများ)၊ portfolio construction (အစုစုတည်ဆောက်ခြင်း)၊ realistic backtesting (လက်တွေ့ကျသော နောက်ပြန်စမ်းသပ်ခြင်း) နဲ့ execution logic (လုပ်ဆောင်မှုယုတ္တိဗေဒ) တွေလည်း လိုအပ်တယ်ဆိုတာကို သိရှိလာကြပါတယ်။ Quant trading ဆိုတာ ညွှန်းကိန်းတစ်ခု (one indicator) ဒါမှမဟုတ် လိမ္မာပါးနပ်တဲ့ idea တစ်ခုတည်းမဟုတ်ပါဘူး။ အဲဒါဟာ အလွှာလိုက်တည်ဆောက်ထားတဲ့ စနစ်တစ်ခုပါ။

ဒီဆောင်းပါးမှာ၊ ကျွန်တော်တို့ဟာ ရိုးရိုး code snippets တွေထက် အများကြီးသာလွန်တဲ့ GitHub repository ၁၀ ခုကို ပြန်လည်သုံးသပ်ခဲ့ပါတယ်။ ၎င်းတို့အားလုံးပေါင်းစပ်ခြင်းအားဖြင့်၊ full frameworks (အပြည့်အစုံမူဘောင်များ)၊ research libraries (သုတေသနစာကြည့်တိုက်များ)၊ structured learning resources (ဖွဲ့စည်းတည်ဆောက်ထားသော သင်ယူမှုအရင်းအမြစ်များ) နဲ့ လက်တွေ့ကုန်သွယ်မှုစနစ်တည်ဆောက်ပုံကို ရောင်ပြန်ဟပ်တဲ့ လက်တွေ့ကိရိယာများကို လွှမ်းခြုံထားပါတယ်။

ဒါတွေကို သေချာစူးစမ်းဖို့ အချိန်ယူမယ်ဆိုရင်၊ ကျပန်း idea တွေကို စမ်းသပ်နေတဲ့သူတစ်ယောက်လိုမဟုတ်ဘဲ၊ ဖွဲ့စည်းတည်ဆောက်ပြီး စည်းကမ်းတကျရှိတဲ့ ကုန်သွယ်မှုလုပ်ငန်းစဉ်ကို ဒီဇိုင်းဆွဲနေတဲ့သူတစ်ယောက်လို တွေးတောလာပါလိမ့်မယ်။ အဲဒီစိတ်နေသဘောထား အပြောင်းအလဲဟာ အပျော်တမ်းစမ်းသပ်မှုတွေနဲ့ လေးနက်တဲ့ quant ဖွံ့ဖြိုးတိုးတက်မှုကို တကယ်ကွဲပြားစေတာပါ။
